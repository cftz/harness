# Cross-Domain Innovation Approach

Use this approach when the problem is **Novel** - unique with no direct solutions, requiring innovative thinking from completely unrelated fields.

## When to Use

```
┌─────────────────────────────────────────────────────────────────────┐
│  ✓ 기존 접근법이 모두 실패한 경우                                   │
│  ✓ 완전히 새로운 제품/서비스 설계                                   │
│  ✓ 기존 시장을 파괴할 혁신이 필요한 경우                            │
│  ✓ 근본적으로 다른 관점이 필요한 경우                               │
│  ✓ 제약 조건이 기존 해결책을 배제하는 경우                          │
└─────────────────────────────────────────────────────────────────────┘
```

## Process

### 1. Deep Abstraction

Remove ALL domain context to find the universal problem:

```
┌─────────────────────────────────────────────────────────────────────┐
│  Level 1: Original Problem                                          │
│  "대규모 분산 시스템에서 효율적인 작업 분배"                         │
│                                                                     │
│                          ▼                                          │
│                                                                     │
│  Level 2: Technical Abstraction                                     │
│  "제한된 리소스를 다수의 요청자에게 공정하게 배분"                   │
│                                                                     │
│                          ▼                                          │
│                                                                     │
│  Level 3: Universal Abstraction                                     │
│  "희소 자원의 효율적이고 공정한 분배"                               │
└─────────────────────────────────────────────────────────────────────┘
```

### 2. Identify Core Tensions

Find the fundamental trade-offs or conflicts:

```
┌─────────────────────────────────────────────────────────────────────┐
│                        Core Tensions                                │
├─────────────────────────────────────────────────────────────────────┤
│  Efficiency    ◄────────────────────────►    Fairness              │
│  Speed         ◄────────────────────────►    Accuracy              │
│  Centralization◄────────────────────────►    Distribution          │
│  Flexibility   ◄────────────────────────►    Consistency           │
│  Individual    ◄────────────────────────►    Collective            │
└─────────────────────────────────────────────────────────────────────┘
```

### 4. WebSearch with Abstract Queries

| Query Template | Example |
|----------------|---------|
| `"how nature solves {abstract-problem}"` | "how nature solves resource distribution" |
| `"biomimicry {abstract-problem}"` | "biomimicry load balancing" |
| `"{field} approach to {abstract-problem}"` | "economics approach to resource allocation" |
| `"{historical-period} solution to {problem}"` | "ancient solution to fair distribution" |

### 5. Extract Universal Principles

For each cross-domain idea:

```
┌─────────────────────────────────────────────────────────────────────┐
│  Source Field         │ What unrelated field is this from?         │
├───────────────────────┼─────────────────────────────────────────────┤
│  Original Application │ How is it used in that field?              │
├───────────────────────┼─────────────────────────────────────────────┤
│  Core Principle       │ What universal law/pattern makes it work?  │
├───────────────────────┼─────────────────────────────────────────────┤
│  Why It Works         │ What fundamental truth does it exploit?    │
├───────────────────────┼─────────────────────────────────────────────┤
│  Innovation Potential │ How might this transform the solution?     │
├───────────────────────┼─────────────────────────────────────────────┤
│  Implementation Sketch│ Initial thoughts on adaptation             │
├───────────────────────┼─────────────────────────────────────────────┤
│  Risks & Unknowns     │ What might not work? What's uncertain?     │
└─────────────────────────────────────────────────────────────────────┘
```

## Output Format

Follow the Output Format defined in `{baseDir}/SKILL.md`.

Use `**Source**:` for the source attribution field (unrelated field where this principle originates).

Additional fields for cross-domain approach:
- **Why This Could Transform the Solution**: How this perspective changes the problem
- **Risks & Unknowns**: What might not work, what needs validation
- **Validation Steps**: How to test if this analogy holds

## Example

**Problem**: "공정하면서도 효율적인 작업 분배 시스템"

**Deep Abstraction**: "희소 자원의 공정하고 효율적인 분배"

**Core Tension**: Efficiency ↔ Fairness

**Cross-Domain Search Results**:

1. **Bee Colony Decision Making**
   - Source: Biology
   - Principle: Waggle dance for democratic resource allocation
   - Application: Decentralized task claiming with quality signals

2. **Auction Theory (Economics)**
   - Source: Economics/Game Theory
   - Principle: Market mechanisms for price discovery
   - Application: Workers bid on tasks based on their efficiency

3. **Triage System (Medicine)**
   - Source: Emergency Medicine
   - Principle: Priority-based allocation under resource constraints
   - Application: Task prioritization with worker capability matching
